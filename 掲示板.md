# 掲示板アプリ 仕様書案　RaptorWorks

## 1. 概要

本システムは、ログイン済みユーザーがMarkdown形式で投稿を作成でき、添付ファイルを安全に扱える掲示板アプリである。
未ログインユーザーは閲覧のみ可能とし、投稿の作成・編集・削除は認証済みユーザーに限定する。

### 1.1 目的

- テキスト中心の情報共有を簡単に行えるようにする
- Markdownにより読みやすい投稿本文を作成可能にする
- 添付ファイルを安全に管理・公開できるようにする

### 1.2 想定利用者

- 一般閲覧ユーザー（未ログイン）
- 投稿ユーザー（ログイン済み）
- 必要に応じて管理者ユーザー

---

## 2. 機能一覧

### 2.1 投稿機能

- 投稿にはタイトルと本文を設定できる
- 本文はMarkdown形式で記述できる
- 投稿の作成・編集・削除はログイン済みユーザーのみ可能

### 2.2 閲覧機能

- 投稿一覧画面で以下を表示する
  - タイトル
  - 投稿者
  - 投稿日時
- 投稿詳細画面で本文をMarkdownレンダリングして表示する
- 投稿詳細画面で添付ファイル一覧を表示する

### 2.3 添付ファイル機能

- 1つの投稿に対して複数ファイルを添付できる
- 添付ファイルは投稿詳細画面から閲覧・ダウンロードできる
- 画像ファイルは画面上でプレビュー表示する
- 画像以外のファイルは以下を表示する
  - ファイル名
  - ファイルサイズ
  - 形式
  - ダウンロードリンク

### 2.4 セキュリティ・制限

- 危険なファイルはアップロード不可とする
- 許可拡張子はホワイトリスト方式で管理する
- 添付ファイルの最大サイズと最大件数を制限する
- Markdown表示時はXSS対策を行い、安全なタグのみ許可する
- 投稿操作には認証が必要とし、未ログインユーザーは閲覧のみ可能とする

## 3. 画面一覧つき仕様書形式

### 3.1 画面一覧

| 画面ID  | 画面名           | 概要                             | 利用者                 |
| ------- | ---------------- | -------------------------------- | ---------------------- |
| SCR-001 | ログイン画面     | ユーザー認証を行う               | 未ログインユーザー     |
| SCR-002 | 投稿一覧画面     | 投稿一覧を表示する               | 全ユーザー             |
| SCR-003 | 投稿詳細画面     | 投稿本文と添付ファイルを表示する | 全ユーザー             |
| SCR-004 | 投稿作成画面     | 新規投稿を作成する               | ログイン済みユーザー   |
| SCR-005 | 投稿編集画面     | 既存投稿を編集する               | 投稿者または権限保持者 |
| SCR-006 | 投稿削除確認画面 | 投稿削除の確認を行う             | 投稿者または権限保持者 |

### 3.2 画面仕様

#### SCR-001 ログイン画面

**概要:** ユーザーID・メールアドレス・パスワード等を用いてログインする。

**主な表示項目:**

- ログインIDまたはメールアドレス
- パスワード入力欄
- ログインボタン
- エラーメッセージ表示欄

**主な動作:**

- 認証成功時、投稿一覧画面へ遷移する
- 認証失敗時、エラーメッセージを表示する

#### SCR-002 投稿一覧画面

**概要:** 登録済みの投稿を一覧表示する。

**表示項目:**

- 投稿タイトル
- 投稿者名
- 投稿日時
- 詳細画面へのリンク
- ログイン済みの場合、新規投稿ボタン

**主な動作:**

- タイトルクリックで投稿詳細画面へ遷移する
- ログイン済みユーザーは新規投稿画面へ遷移できる

**一覧表示ルール:**

- 新しい投稿を上位に表示する
- 投稿日時は日時形式で表示する
- 必要に応じてページネーションまたは無限スクロールを導入する

#### SCR-003 投稿詳細画面

**概要:** 投稿本文と添付ファイルを表示する。

**表示項目:**

- タイトル
- 投稿者名
- 投稿日時
- 本文（Markdownレンダリング後）
- 添付ファイル一覧
- 編集ボタン
- 削除ボタン

**添付ファイル表示仕様:**

画像ファイルの場合:

- 対象例: jpg, jpeg, png, gif, webp
- 画面上にプレビュー表示する
- 原寸または縮小表示を行う
- クリックで別タブ表示またはダウンロードできる

画像以外のファイルの場合:

- ファイル名
- ファイルサイズ
- MIMEタイプまたは拡張子
- ダウンロードリンク

**ボタン制御:**

- 編集・削除ボタンはログイン済みかつ権限保持者のみ表示する
- 未ログインユーザーには表示しない

#### SCR-004 投稿作成画面

**概要:** 新規投稿を作成する。

**入力項目:**

- タイトル
- 本文（Markdown入力）
- 添付ファイル選択欄（複数選択可）

**バリデーション:**

- タイトル: 必須
- 本文: 必須
- 添付ファイル数: 最大5件
- 1ファイルサイズ: 最大10MB
- 許可拡張子のみアップロード可

**主な動作:**

- 登録成功時、投稿詳細画面へ遷移する
- 入力不備時、エラーメッセージを表示する

#### SCR-005 投稿編集画面

**概要:** 既存投稿を編集する。

**入力項目:**

- タイトル
- 本文（Markdown入力）
- 添付ファイル追加
- 既存添付ファイル削除指定

**主な動作:**

- 投稿者本人または権限者のみ編集可能
- 更新成功時、投稿詳細画面へ遷移する

#### SCR-006 投稿削除確認画面

**概要:** 投稿削除前の確認を行う。

**表示項目:**

- タイトル
- 削除確認メッセージ
- 削除実行ボタン
- キャンセルボタン

**主な動作:**

- 削除実行後、投稿一覧へ遷移する
- 添付ファイルも論理削除または物理削除の方針に従って処理する

## 4. 業務ルール・制約

### 4.1 投稿ルール

- 投稿作成・編集・削除はログイン済みユーザーのみ可能
- 投稿閲覧は未ログインユーザーでも可能
- 編集・削除は原則として投稿者本人のみ可能（ただし管理者権限がある場合は例外とする）

### 4.2 添付ファイルルール

- 1投稿あたり最大5件
- 1ファイルあたり最大10MB
- 複数ファイル添付可能
- ホワイトリストにない拡張子は拒否する

### 4.3 許可ファイル例

`pdf`, `txt`, `docx`, `xlsx`, `jpg`, `jpeg`, `png`, `gif`, `webp`

### 4.4 禁止ファイル例

`exe`, `bat`, `cmd`, `scr`, `ps1`, `js`, `vbs`

### 4.5 Markdown表示ルール

- Markdown本文はHTMLへ変換して表示する
- 表示時はサニタイズ処理を実施する
- scriptタグや危険な属性は除去する
- 許可するタグのみ描画対象とする

---

## 5. 非機能要件

### 5.1 セキュリティ

- 認証されていないユーザーは投稿操作不可
- 投稿更新・削除時は認可チェックを行う
- Markdownレンダリング時はXSS対策を行う
- 添付ファイルアップロード時は拡張子・MIMEタイプ・サイズを検証する
- 危険ファイルは保存前に拒否する

### 5.2 可用性・運用

- 添付ファイルの保存先はローカルまたはクラウドストレージを想定
- ファイル削除ポリシーは論理削除または物理削除を別途定義する
- 投稿・添付ファイルの監査ログを必要に応じて保持する

### 5.3 性能

- 投稿一覧はページネーション対応を推奨
- 大きな画像はサムネイル化または表示最適化を推奨
- ファイルダウンロードはストリーミング配信を推奨

## 7. API仕様ベース

以下はREST APIを前提とした仕様案。

### 7.1 認証

**`POST /api/auth/login`** — ログインを行う。

リクエスト例:

```json
{
  "email": "user@example.com",
  "password": "password"
}
```

レスポンス例:

```json
{
  "token": "jwt-token",
  "user": {
    "id": "u_001",
    "name": "山田太郎"
  }
}
```

### 7.2 投稿一覧取得

**`GET /api/posts`** — 投稿一覧を取得する。

クエリパラメータ例: `page`, `perPage`, `keyword`

レスポンス例:

```json
{
  "items": [
    {
      "id": "p_001",
      "title": "お知らせ",
      "author": {
        "id": "u_001",
        "name": "山田太郎"
      },
      "createdAt": "2026-04-02T10:00:00Z"
    }
  ],
  "page": 1,
  "perPage": 20,
  "total": 100
}
```

### 7.3 投稿詳細取得

**`GET /api/posts/{postId}`** — 投稿詳細を取得する。

レスポンス例:

```json
{
  "id": "p_001",
  "title": "お知らせ",
  "bodyMarkdown": "# 見出し\n本文です。",
  "bodyHtml": "<h1>見出し</h1><p>本文です。</p>",
  "author": {
    "id": "u_001",
    "name": "山田太郎"
  },
  "attachments": [
    {
      "id": "f_001",
      "fileName": "sample.pdf",
      "fileSize": 102400,
      "mimeType": "application/pdf",
      "extension": "pdf",
      "downloadUrl": "/api/files/f_001/download",
      "previewUrl": null,
      "isImage": false
    },
    {
      "id": "f_002",
      "fileName": "image.png",
      "fileSize": 204800,
      "mimeType": "image/png",
      "extension": "png",
      "downloadUrl": "/api/files/f_002/download",
      "previewUrl": "/api/files/f_002/preview",
      "isImage": true
    }
  ],
  "createdAt": "2026-04-02T10:00:00Z",
  "updatedAt": "2026-04-02T10:30:00Z"
}
```

### 7.4 投稿作成

**`POST /api/posts`** — 新規投稿を作成する。認証必須。

リクエスト例:

```json
{
  "title": "新規投稿タイトル",
  "bodyMarkdown": "# 本文\nMarkdownで記述します。",
  "attachmentIds": ["f_tmp_001", "f_tmp_002"]
}
```

レスポンス例:

```json
{
  "id": "p_002",
  "message": "Post created successfully"
}
```

**バリデーション:**

- `title`: 必須、最大文字数を設定
- `bodyMarkdown`: 必須
- `attachmentIds`: 任意、最大5件

### 7.5 投稿更新

**`PUT /api/posts/{postId}`** — 投稿を更新する。認証必須、かつ投稿者本人または権限保持者のみ。

リクエスト例:

```json
{
  "title": "更新後タイトル",
  "bodyMarkdown": "更新後の本文",
  "attachmentIds": ["f_003", "f_004"],
  "deleteAttachmentIds": ["f_001"]
}
```

レスポンス例:

```json
{
  "message": "Post updated successfully"
}
```

### 7.6 投稿削除

**`DELETE /api/posts/{postId}`** — 投稿を削除する。認証必須、かつ投稿者本人または権限保持者のみ。

レスポンス例:

```json
{
  "message": "Post deleted successfully"
}
```

### 7.7 添付ファイルアップロード

**`POST /api/files`** — 添付ファイルをアップロードする。認証必須。

**仕様:**

- `multipart/form-data` を使用
- 一時アップロード後、投稿作成時に紐付ける運用を想定

リクエスト: `file`

レスポンス例:

```json
{
  "id": "f_tmp_001",
  "fileName": "document.pdf",
  "fileSize": 512000,
  "mimeType": "application/pdf",
  "extension": "pdf",
  "isImage": false
}
```

**バリデーション:**

- 許可拡張子のみ許可
- 最大サイズ10MB
- 危険ファイルは拒否
- MIMEタイプと拡張子の整合性チェックを推奨

### 7.8 添付ファイルダウンロード

**`GET /api/files/{fileId}/download`** — 添付ファイルをダウンロードする。

**動作:**

- 権限のあるユーザー、または公開閲覧可能な投稿に紐づく場合のみ取得可
- `Content-Disposition` を付与してダウンロードさせる

### 7.9 添付ファイルプレビュー

**`GET /api/files/{fileId}/preview`** — 画像ファイルをプレビュー表示する。

**対象:** `jpg`, `jpeg`, `png`, `gif`, `webp`

**動作:**

- 画像ファイルのみレスポンス対象
- 非画像ファイルに対してはエラーを返す

## 8. APIエラー仕様案

### 共通エラーレスポンス

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "入力内容に誤りがあります",
    "details": [
      {
        "field": "title",
        "message": "タイトルは必須です"
      }
    ]
  }
}
```

### 主なエラーコード例

| コード                | 内容                         |
| --------------------- | ---------------------------- |
| UNAUTHORIZED          | 未認証                       |
| FORBIDDEN             | 権限不足                     |
| NOT_FOUND             | 対象データなし               |
| VALIDATION_ERROR      | 入力値不正                   |
| FILE_TYPE_NOT_ALLOWED | 許可されていないファイル形式 |
| FILE_SIZE_TOO_LARGE   | ファイルサイズ超過           |
| FILE_COUNT_EXCEEDED   | 添付数上限超過               |

## 9. 補足設計方針

### 9.1 Markdown

- 保存時はMarkdown原文を保持する
- 表示時にHTMLへ変換する
- HTML変換後はサニタイズする

### 9.2 添付ファイル

- ファイル名の重複を避けるため保存名は内部で別名管理する
- 元ファイル名は表示用として保持する
- 公開URL直置きではなく、API経由配信を推奨する

### 9.3 認可

- 投稿編集・削除可否はサーバー側で必ず判定する
- フロント側のボタン非表示だけでは制御完了としない
